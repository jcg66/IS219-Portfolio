Premise

As AI accelerates code generation, the primary bottleneck in software engineering shifts from production to validation. This project engineered a data pipeline to track this "Role-Mix Shift," providing engineering managers with empirical evidence of how responsibilities are migrating toward review and oversight.

The "Builder" Revision (V2.0 Updates)

To align with a professional systems engineering identity, the project was overhauled:

  • From Narrative to Dashboard: Replaced long-form text with a "Command Center" UI, prioritizing density and data-clarity.
  • Data Provenance focus: Built an explicit /verification workspace to smoke-test API and MCP pathways, ensuring the AI insights are grounded in raw JSON data (Stack Overflow, GitHub Innovation Graph, and NVD).
  • Architecture over Content: Framed the code not as a website, but as a Data Pipeline that normalizes mixed-granularity sources (Annual Surveys vs. Real-time CVE feeds) into a unified analysis schema.

Key Engineering Features

  • Normalization Engine: Scripts in lib/data/ that align disparate time-series data (2019–2024) into a consistent YYYY-Qn frame.
  • Grounded AI Interaction: An AI Chat interface utilizing MCP-backed data pathways, preventing hallucination by forcing the model to query local project datasets.
  • Multi-Source Correlation: Dynamic visualization comparing ai_tool_usage_pct against vuln_rate_per_kloc to test the hypothesis that increased speed without increased oversight elevates risk.

Outcome

The project successfully demonstrates a "Reviewer-Heavy" trendline. It proves that while code volume is increasing, the demand for human-in-the-loop validation is growing faster, justifying the need for specialized "AI Oversight" roles.
